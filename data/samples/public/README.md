# data/samples/public/ — 공개 샘플 문서 3개

## 목적

`/qa` 자동화 시나리오 (매뉴얼 §3.5-9) 와 `feature/project-shell` 업로드 플로우 검증에서
**재현 가능한 공개 자료**를 사용해야 한다. private 샘플 (`data/samples/private/`) 은 절대 쓰지 않는다.

## 법적 주의

- 여기에는 **KOGL(공공누리) 또는 동등한 공개 라이선스**로 제공되는 자료만 넣는다.
- EIASS 원문·비-KOGL 자료의 재배포는 `CLAUDE.md §2-4` 위반이다.
- 다운로드한 파일의 출처(URL, 라이선스)를 아래 표에 기록한다.

## 필요 파일 (QA 시나리오 기준)

매뉴얼 §3.5-9 시나리오 4 "민감 업로드 흐름" 이 통과하려면 **3개의 공개 샘플**이 필요하다:

| 파일명 | 용도 | 포맷 | 크기 권장 |
|---|---|---|---|
| `sample_minimal.pdf` | 정상 업로드·삭제 경로 | PDF | 100KB–2MB |
| `sample_docx.docx` | DOCX 파서 smoke test | DOCX | 100KB–2MB |
| `sample_large.pdf` | 50MB 근처 상한 테스트 | PDF | 20–49MB |

## 조달 방법

아래 중 택1 로 3개 파일을 확보한 뒤 이 폴더에 넣는다:

### 옵션 A — 공공 환경영향평가 간이자료 (추천, 1인 개발 기준)
1. 환경부·환경정책평가연구원(KEI) 공개 보고서 중 **KOGL 1유형** 으로 표시된 PDF 1건.
2. 공공기관 공개 보도자료 PDF 1건 (위와 다른 기관, KOGL 표시 확인).
3. 위 중 하나를 DOCX 로 변환(Word 사용)하여 `sample_docx.docx` 로 저장.

### 옵션 B — 이 저장소에서 직접 만든 mock 문서
1. `docs/` 의 더미 마크다운을 PDF 로 인쇄 → `sample_minimal.pdf`.
2. 같은 문서를 DOCX 로 저장 → `sample_docx.docx`.
3. 50MB 근처 더미 PDF 를 만들어 업로드 상한 테스트.

옵션 B 가 저작권 리스크가 가장 낮다. 옵션 A 는 실제 평가사 시나리오에 더 가깝다.
