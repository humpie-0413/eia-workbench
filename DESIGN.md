# DESIGN.md — eia-workbench 디자인 시스템 (v0 초안)

> 본 초안은 매뉴얼 §9.1~9.2 의 규칙을 기반으로 만들었다.
> `/design-consultation` 을 한 번 더 돌려 v1 로 확정한다.

## 0. 본 프로젝트 디자인 미션

1인 평가대행사 실무자가 **산만함 없이 검수 작업에 집중**할 수 있게 한다.
정보는 밀도 있게, 결정은 신중하게, 톤은 단정하지 않게.

## 1. 톤앤매너

### 1.1 사용하지 않는 것
- 이모지 (UI·카피·에러메시지 전부)
- 그라데이션, 네온, 글래스모피즘, 파스텔 배경
- 3색 이상의 강조색
- 일러스트풍 일러스트·만화적 스타일
- "승인", "통과", "확정", "대상입니다" 같은 법적 단정 표현
- 과도한 애니메이션 (150ms 이내만 허용)

### 1.2 쓰는 것
- 중립적인 타이포그래피 (Inter 또는 Pretendard)
- 1 Primary + 1 Warning + Neutral 스케일만
- "근거 / 가정 / 한계 / 사람 검토 필요" 4필드가 결과 카드에 항상 노출
- "가능성", "확인 필요", "전문가 검토 필요" 같은 조건부 언어

**금지:** 그라데이션 배경, 브랜드 외 원색, 파스텔 톤 배경 카드.

### 2.1 스코핑 결과 배지 토큰 (feature/scoping-assistant-v2)

5종 카테고리·상태 배지. 텍스트·배경 조합은 모두 WCAG AA 이상(≥ 4.5:1) 대비 검증.

| 토큰 | 용도 | 텍스트 / 배경 | 대비 |
|---|---|---|---|
| badge `eia_target` triggered | "본 평가 대상 가능성" | `#991B1B` / `#FEE2E2` | 6.33:1 |
| badge `small_eia` triggered | "소규모 평가 대상 가능성" | `#9A3412` / `#FFEDD5` | 6.15:1 |
| badge `forest_conversion` triggered | "산지전용 협의 필요 가능성" | `#166534` / `#DCFCE7` | 6.87:1 |
| badge `needs_human_review` | "사람 검토 필요" (항상) | `#1F2937` / `#E5E7EB` | 10.5:1 |
| badge skip (`input_undefined` / `zone_mismatch` / `condition_not_met`) | 스킵 사유 | `#52525B` / `#F4F4F5` | 7.8:1 |

**사용 규칙:**
- triggered=true 인 rule 만 원색 배지. skip/false 는 중성 배지(skip 토큰).
- "사람 검토 필요" 배지는 모든 결과 카드에 동시 노출. 이 배지는 `aria-label="사람 검토 필요 표시"`.
- 배지 라운드 4px, padding 4px/8px, 텍스트 13px/600.

## 4. 간격·레이아웃

- 기본 단위: 4px 스케일 (`4, 8, 12, 16, 24, 32, 48`)
- 컨텐트 최대 폭: 1200px (검수 작업 기준)
- 카드 안쪽 여백: 24px / 모바일 16px
- 섹션 간격: 32px
- 테이블 행 높이: 36px 이상 (조밀하게)

## 5. 컴포넌트 기본

### 5.1 Button
- Primary, Secondary, Danger, Ghost 네 종류
- 라운드 6px, 높이 36px, Medium 14px/600
- Disabled 는 opacity 50% + cursor not-allowed
- 아이콘 포함 시 좌 8px 간격

### 5.2 Form Field
- Label 항상 시인성 있게 (placeholder only 금지)
- 에러 메시지는 필드 바로 아래, 색은 `--c-error`
- 필수 표시: 라벨 뒤 `*`, `aria-required="true"`

### 5.3 결과 카드 (eia-workbench 특화)
모든 분석·체크 결과 표시에 사용. **4개 필드 필수:**

```
┌─────────────────────────────────────────────────────┐
│ 제목                                    [재검토]   │
│ ─────────────────────────────────────────────────── │
│ 결과:   이 항목은 확인이 필요한 가능성이 있습니다 │
│ 근거:   사용자가 입력한 대상사업 면적 X ha 기준 … │
│ 가정:   육상풍력·환경영향평가법 시행령 별표1 적용 │
│ 한계:   현지조사는 대체하지 않는다. 사용자 확인 필요│
│ [사람 검토 필요]  ← 항상 노출되는 배지             │
└─────────────────────────────────────────────────────┘
```

4개 필드 중 하나라도 빠지면 **렌더 거부 + 콘솔 경고**.

### 5.4 Table
- 헤더 행 `--c-text-secondary`, 본문 `--c-text-primary`
- 행 hover 배경 `--c-bg` 2% 진한 색
- 빈 상태는 "데이터가 없습니다" 한 줄 + 도움말 링크

### 5.5 Modal / Dialog
- 오버레이 `rgba(0,0,0,0.4)`
- Esc / 오버레이 클릭으로 닫기
- 포커스 트랩, 닫기 시 원래 포커스 복원

## 6. 접근성

- 모든 인터랙티브 요소에 포커스 링 필수 (`:focus-visible`)
- 대비 AA 이상 (`--c-text-primary` / `--c-bg` > 7:1)
- 키보드 전용 조작 가능 (Tab 순서, Enter/Space로 모든 액션)
- 폼: `<label for>` 또는 `aria-label`
- 결과 카드의 "사람 검토 필요" 배지는 `aria-label="사람 검토 필요 표시"`
- 이미지·아이콘 `alt` 또는 `aria-hidden`

## 7. 법적 단정 표현 자동 검출

테스트 계층에서 다음 정규식에 매칭되는 UI 문자열이 있으면 fail:

```
/(환경영향평가\s*대상입니다|협의\s*통과|승인됨|법적으로\s*문제\s*없|자동.*(완료|작성).*(안전|문제없))/
```

대체 표현:
- "대상입니다" → "대상 가능성이 있어 전문가 확인이 필요합니다"
- "통과/승인" → "보완 리스크를 낮추는 체크 항목입니다"
- "AI가 현황조사를 완료했습니다" → "문헌·입력자료 기준 검토이며 현지조사는 별도 필요합니다"
