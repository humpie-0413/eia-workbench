# DESIGN.md — eia-workbench 디자인 원칙 (초안)

> 이 파일은 `/design-consultation` 을 실제로 돌려 확정한다. 아래는 스캐폴딩용 초안 가드.

## 톤앤매너
- 업무용 SaaS. 단정 표현 금지 ("승인/통과/대상입니다" 등).
- 이모지 금지. 일러스트·그라데이션·네온·글래스모피즘·파스텔 배경 금지.
- 결과 카드에는 `근거 / 가정 / 한계 / 사람 검토 필요` 4필드를 항상 노출.

## 색 시스템 (임시)
- Primary 1색, Neutral 1스케일, 경고 1색. 강조색 3개 이상 금지.

## 타이포
- 본문 15–16px, 라인하이트 1.55 기준. 제목은 수직 여백으로 분리.

## 밀도
- 데스크톱 1280px 기준, 핵심 정보 3블록 이상 한 화면.

## 접근성
- axe 기본 체크 통과, 키보드 내비, 포커스 링 유지, 대비 AA 이상.

1. `/design-consultation` 실행 → 본 파일 확정.
2. 컴포넌트 목록과 토큰을 `docs/design/` 에 분리.
