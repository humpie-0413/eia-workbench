### 🔴 Finding 2: 환경영향평가 대상 MW 임계값 **10배 오류** — CRITICAL

| 항목 | 초안 spec §7 rule 1 | 현행 법령 (2차 자료 기준) |
|---|---|---|
| 조건 | `capacity_mw >= 10` → 대상 가능성 | 발전시설용량 **100MW (10만 킬로와트)** 이상 |
| 단위 | MW | kW (10만 kW = 100 MW) |

**근거:**
- 2차 자료 인용 (WebSearch 검색 결과): "태양력·풍력, 연료전지발전소 또는 발전사업용 전기저장장치의 경우에는 발전시설용량이 **10만킬로와트 이상**인 것이 환경영향평가 대상입니다."
- 환경영향평가법 시행령 별표3 (2차 자료 요약 기준): "100MW 이상, 사업면적 30만㎡ 이상인 풍력발전에 적용"

**영향:**
- spec rule 1 이 그대로 구현되면 **10MW 짜리 육상풍력** (실제로는 법적 EIA 대상 아님) 도 시스템이 "대상 가능성" 이라고 표시함.
- 사용자(환경영향평가대행사) 관점에서 **치명적 오판** — 소규모환경영향평가만 필요한 사업을 전체 EIA 로 착수하면 수개월의 불필요한 절차.
- CLAUDE.md §9.2 "법적 결론 단정 표현 금지" 에 직접 위반되는 수치.

### 🔴 Finding 3: 1–10 MW 구간과 0.1–1 MW 구간의 기준 축 오류 — HIGH

| 항목 | 초안 spec §7 rule 2 / rule 3 | 현행 법령 (2차 자료 기준) |
|---|---|---|
| 1–10 MW | 소규모 환경영향평가 대상 (발전용량 기준) | 소규모 EIA 대상은 **발전용량이 아닌 사업계획 면적** 기준 (용도지역별로 다름) |
| 0.1–1 MW | 검토 필요 (발전용량 기준) | 소규모 EIA 는 발전용량이 아니라 면적이 핵심 축. 극소규모는 별도 조항 필요. |

**근거:**
- 2차 자료 인용: "보전관리지역의 경우 **사업계획 면적이 5,000㎡ 이상**인 경우가 소규모 환경영향평가 대상사업"
- 환경영향평가법 시행령 별표4 는 "지역별(보전/생산/계획관리지역, 농림지역, 자연환경보전지역 등) + 사업면적" 조합으로 테이블이 구성됨 — MW 기준이 주축이 아님.

**영향:**
- spec §5 입력 스키마는 `capacity_mw` 위주로 구성돼 있어 **법령이 실제로 요구하는 입력(사업면적 × 용도지역)** 이 스키마에 없음.
- 구현 전 **입력 스키마 재설계** 필요. `site_area_m2` 및 `land_use_zone`(보전관리/생산관리/계획관리/농림/자연환경보전) 필드 추가 검토.

### 🟠 Finding 4: 산림 전용 임계값 축 오류 — HIGH

| 항목 | 초안 spec §7 rule 4 | 현행 법령 (2차 자료 기준) |
|---|---|---|
| 조건 | `forest_conversion_ha > 1` → 추가 검토 | 풍력발전시설은 **660㎡(0.066ha) 이상** 이면 산지전용타당성조사 대상 |
| 근거 법령 | 산지관리법 시행령 | 산지관리법 + 산지관리법 시행령 + 시행규칙 별표4 |

**근거:**
- 2차 자료 인용: "풍력발전시설 또는 궤도시설을 설치하는 경우 **660㎡ 이상**으로 산지전용허가 또는 산지일시사용허가를 받으려는 자는 미리 **산지전용타당성조사**를 받아야 합니다."
- 태양광발전은 "허가면적을 3만제곱미터 이상으로 할 수 없다" 와 같이 별도 상한도 있음 (풍력은 아님).

**영향:**
- 초안의 1 ha 기준은 법정 임계값(660㎡ = 0.066 ha) 보다 **약 15배 완화된 수치** — 실제로는 산지전용타당성조사가 필요한 사업을 시스템이 "추가 검토 없음" 으로 넘길 위험.
- 단, 이것은 "환경영향평가 대상 여부" 와 별개인 "산지관리법상 추가 절차" 이므로, rule pack 의 분류(`result: needs_review`) 자체는 합리적. 임계값만 교정하면 됨.

## 3. Verdict

**FAIL — CRITICAL** (4건 중 2건 CRITICAL, 2건 HIGH)

Spec §7 을 그대로 구현하면:
1. `별표2` → `별표3` 으로 교정되지 않은 채 사용자 UI 에 잘못된 법령 근거가 노출됨
2. 10 MW 임계값이 10배 잘못돼 **실제 법적 대상이 아닌 사업을 "대상 가능성" 으로 표시**
3. 소규모 EIA 판단 축(면적 × 용도지역)이 입력 스키마에서 누락돼 **법적으로 무의미한 결과** 생성
4. 산지관리법 660㎡ 기준이 1ha 로 완화돼 산지전용타당성조사 대상 사업을 놓침

### A안 (권장): 입력 스키마·rule pack 재설계 + 원문 확보 후 재착수

### B안 (최소 교정): 숫자만 교정하고 MVP 축소

**이유:**

- 단, UI 면에 "본 도구는 **발전용량만을 기준으로** 개략 스크리닝하며, 소규모 환경영향평가는 실제로는 면적·용도지역 기준이므로 **반드시 전문가 검토 필요**" 라는 명시적 고지가 필요.

**필요 작업:**
1. spec §7:
   - rule 1: `capacity_mw >= 100` (10 → 100), citation `별표3` (별표2 → 별표3)
   - rule 2: 삭제 또는 "1–100 MW 구간은 면적·용도지역 기준으로 별도 판단 필요" 로 단일화
   - rule 3: 삭제
   - rule 4: `forest_conversion_ha > 0.066` (1 → 0.066), citation `산지관리법 시행령 + 시행규칙 별표4`

3. rule 개수가 2 개로 줄어들면 결과 UX 재검토 필요.

**예상 추가 공수:** 2~4 h (숫자·citation 교체 + UI 배너 재설계).
