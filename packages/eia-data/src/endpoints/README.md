# 엔드포인트 구현 가이드

이 디렉터리에는 data.go.kr 의 각 데이터셋별 엔드포인트 구현이 들어간다.

**현재는 비어있다.** 다음 feature (유사사례 검색) 에서 필요 데이터셋부터 채운다.

## 구현 규칙

1. 파일명: `<카테고리><ID>-<kebab-name>.ts` (예: `a15028236-air-quality.ts`, `b15000800-eia-info.ts`)
2. 각 파일은 다음을 export:
   - 요청 타입: `XxxRequest`
   - 응답 항목 타입: `XxxItem`
   - 호출 함수: `fetchXxx(client: PortalClient, req: XxxRequest): Promise<XxxItem[]>`
3. 결과는 프로젝트 §5 표준 스키마로 변환해 UI 에 전달: `{result, basis, assumptions, limits, needsHumanReview}`
4. 응답이 XML 이면 파싱 유틸을 먼저 추가 (의존성은 협의 후 루트에 add).
5. 일 호출 한도를 고려해 같은 요청은 캐시 (D1 or edge cache).

## 공통 제약

- 일 호출 한도 존재 (개발계정 1,000건/일 전후). 캐시 전략 필수.
- 응답 포맷 XML/JSON 혼재. 엔드포인트별 확인.
- SERVICE_KEY 는 Cloudflare Secret 으로 주입. 절대 커밋 금지.
- 출처 표시 유지: "공공데이터포털 (data.go.kr), 제공: 한국환경연구원".
